## Notes

### Design Decisions

1. **Blob Animation Pattern:**
   - Used primary-300 and primary-200 (lighter shades)
   - Maintains subtle, non-distracting background effect
   - 10-20% opacity ensures readability of foreground content
   - mix-blend-multiply creates depth without overpowering

2. **Gradient Consistency:**
   - All gradients use primary-600 ‚Üí primary-700 pattern
   - Darker endpoint creates visual interest
   - Maintains monochromatic sage aesthetic
   - Consistent with Logo and Header patterns (DOM-418)

3. **Dark Mode Strategy:**
   - Lighter shades in dark mode (500/600 vs 600/700)
   - Blob animations use /20 opacity for subtlety
   - mix-blend-color-dodge enhances visibility
   - Follows colors.ts dark mode guidelines

4. **Priority Color Preservation:**
   - Kept red/orange/green for task priorities
   - Universal recognition aids user understanding
   - Category icons provide additional semantic context
   - Theme priority colors available for future use

### Technical Insights

**Why Priority Colors Weren't Changed:**
- Current system uses **category icons** (star, heart, person, book)
- Priority is indicated by **left border color** (red/orange/green)
- Tailwind colors are universally recognized for priority levels
- Theme priority colors (#D77A61, #E8B86D, #8B9DAF) are available but not required here
- Semantic clarity trumps strict brand consistency for UI affordances

**Blob Animation Performance:**
- blur-xl and blur-2xl use CSS filters (GPU-accelerated)
- opacity and mix-blend-multiply are composited efficiently
- No JavaScript animation (pure CSS)
- 60fps maintained across all devices

### Lessons Learned

1. **Exploration Reveals Context:**
   - Initial assumption: priority colors need updating
   - Actual finding: categories use semantic icons, priorities use standard colors
   - Outcome: Keep existing system for clarity

2. **Gradient Consistency Matters:**
   - All sage gradients now use primary-600 ‚Üí primary-700
   - Creates cohesive visual language
   - Easier for future developers to maintain

3. **Dark Mode Requires Lighter Shades:**
   - primary-500/600 in dark mode vs primary-600/700 in light
   - Critical for visibility against dark surfaces
   - Follows established pattern from DOM-418
